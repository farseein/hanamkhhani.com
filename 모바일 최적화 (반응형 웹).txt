홈페이지를 모바일 최적화로 바꾸는 건 어떻게 코딩해야돼?



홈페이지를 모바일 최적화(반응형 웹)로 만드는 것은 현대 웹 개발의 기본이자 필수입니다. 사용자의 화면 크기에 따라 디자인이 유연하게 변하도록 만드는 핵심 기술들을 정리해 드릴게요.

1. 뷰포트(Viewport) 메타 태그 설정
모바일 브라우저에게 "이 페이지는 화면 너비에 맞춰서 보여줘"라고 알려주는 가장 첫 번째 단계입니다. HTML의 <head> 태그 안에 아래 코드를 반드시 넣어야 합니다.

HTML
<meta name="viewport" content="width=device-width, initial-scale=1.0">
2. 미디어 쿼리(Media Queries) 사용
CSS에서 특정 화면 너비 이하일 때만 적용할 스타일을 정의합니다. 보통 스마트폰, 태블릿, 데스크톱의 기준점(Breakpoint)을 나눕니다.

CSS
/* 기본은 모바일 스타일로 작성 (Mobile First) */
body { font-size: 14px; }

/* 태블릿 이상 (768px 이상) */
@media (min-width: 768px) {
  body { font-size: 16px; }
}

/* 데스크톱 이상 (1024px 이상) */
@media (min-width: 1024px) {
  body { font-size: 18px; }
}
3. 유연한 레이아웃 (Flexbox & Grid)
과거에는 고정된 픽셀(px)을 썼지만, 모바일 최적화를 위해서는 비율이나 유연한 단위를 사용해야 합니다.

가변 너비: width: 500px; 대신 width: 100%; 또는 max-width: 100%;를 사용하세요.

Flexbox: 요소를 가로/세로로 쉽게 배치하고, 공간이 부족하면 자동으로 줄바꿈(flex-wrap: wrap)되게 만듭니다.

4. 이미지 최적화
이미지가 화면 밖으로 튀어나가지 않도록 CSS에 아래 설정을 추가하는 것이 좋습니다.

CSS
img {
  max-width: 100%;
  height: auto;
}
💡 모바일 최적화 체크리스트
터치 영역 확보: 버튼이나 링크는 손가락으로 누르기 쉽게 최소 44x44px 이상의 크기를 권장합니다.

폰트 가독성: 모바일에서는 최소 16px 이상의 글자 크기가 읽기 편합니다.

가로 스크롤 금지: 사용자가 페이지를 옆으로 밀어야 한다면 최적화에 실패한 것입니다.

기존에 작성하신 코드 중에 특히 모바일에서 깨져 보이는 부분이 있나요? 그 부분의 HTML/CSS 코드를 보여주시면 제가 직접 수정해 드릴 수 있습니다. 혹은 특정 브레이크포인트 설정이 궁금하신가요?
